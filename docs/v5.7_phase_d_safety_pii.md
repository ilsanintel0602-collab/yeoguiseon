## 1. 위험물 안전 UI (danger=true 빨간 경고)

### 현재 상태

`SYSTEM_PROMPT`에 `danger=true` 명시:
- 가스통/부탄캔/스프레이 = hazardous_gas
- 부풀은 리튬배터리 = hazardous_battery
- 주사기/환자명 약통 = hazardous_medical
- 깨진 유리·칼날 = sharp_glass

근데 모바일 결과 화면에서 **일반 결과 카드와 똑같이** 표시됨. 위험성 강조 부족.

### 패치 — `app.html`의 결과 렌더링 함수에 추가

// 결과 렌더링 분기 추가
function renderResult(result) {
  if (result.danger === true || result.category_hint === 'hazardous') {
    return renderDangerWarning(result) + renderNormalCard(result);  // 경고 위에 + 일반 카드 아래
  }
  return renderNormalCard(result);
}

## 2. PII 자동 블러 (영수증·문서 개인정보 보호)

### 시나리오

사용자가 영수증 촬영 → 사진에 본명·전화·주소 노출 → Gemini로 전송 시 외부 노출 위험.

### 흐름

```
[촬영] → [캔버스 OCR로 텍스트 영역 감지] → [개인정보 패턴 매치] → [블러 처리]
       ↓
       [블러된 이미지 미리보기 + 사용자 확인]
       ↓
       Y → 블러 이미지 전송
       N → 원본 전송 (사용자 선택)
```

### 코드 (app.html 분석 함수 직전)

```javascript
// v5.7 Phase D: PII 자동 감지 + 블러
async function autoBlurPII(imageDataUrl) {
  // Tesseract.js로 한국어+영문 텍스트 영역 감지
  const worker = await loadOcrWorker();
  const result = await worker.recognize(imageDataUrl);
  
  // 개인정보 패턴
  const PII_PATTERNS = [
    /\d{3}-\d{4}-\d{4}/g,                            // 전화번호 010-1234-5678
    /\d{2,3}-\d{3,4}-\d{4}/g,                        // 일반전화
    /\d{6}-\d{7}/g,                                  // 주민등록번호
    /[가-힣]{2,4}\s*(님|씨|선생님)/g,                  // 이름 호칭
    /[A-Z][a-z]+ [A-Z][a-z]+/g,                      // 영문 이름
    /[가-힣]{1,3}\s*[시군구]\s*[가-힣]+\s*[동읍면리]/g, // 한국 주소
  ];
  
  // 텍스트 영역 중 PII 매치되는 곳만 추출
  const piiBoxes = [];
  for (const line of result.data.lines) {
    for (const pattern of PII_PATTERNS) {
      if (pattern.test(line.text)) {
        piiBoxes.push(line.bbox); // {x0, y0, x1, y1}
        break;
      }
    }
  }
  
  if (piiBoxes.length === 0) return imageDataUrl; // PII 없음, 원본 그대로
  
  // 캔버스에 그리고 PII 영역 블러
  const img = new Image();
  await new Promise(res => { img.onload = res; img.src = imageDataUrl; });
  const canvas = document.createElement('canvas');
  canvas.width = img.width;
  canvas.height = img.height;
  const ctx = canvas.getContext('2d');
  ctx.drawImage(img, 0, 0);
  
  // 각 PII box를 가우시안 블러 (filter blur(20px))
  for (const box of piiBoxes) {
    ctx.filter = 'blur(20px)';
    ctx.drawImage(canvas, 
      box.x0, box.y0, box.x1 - box.x0, box.y1 - box.y0,
      box.x0, box.y0, box.x1 - box.x0, box.y1 - box.y0
    );
    ctx.filter = 'none';
  }
  
  return canvas.toDataURL('image/jpeg', 0.92);
}

// 분석 시작 직전 호출
async function analyzeWithPIIProtection(imageDataUrl) {
  toast('🔒 개인정보 자동 확인 중...', false);
  const blurredImage = await autoBlurPII(imageDataUrl);
  
  if (blurredImage !== imageDataUrl) {
    // PII 감지됨 — 사용자 확인
    const confirmed = confirm(
      '⚠️ 영수증·문서에서 개인정보 발견.\n자동 블러 후 전송할까요?\n\n[확인] = 블러 후 전송 (권장)\n[취소] = 원본 전송 (개인정보 노출 가능)'
    );
    if (confirmed) {
      toast('✅ 개인정보 블러 완료, 분석 시작', false);
      return analyzeHybrid(blurredImage);
    } else {
      toast('⚠️ 원본 전송 — 개인정보 외부 노출 가능', true);
      return analyzeHybrid(imageDataUrl);
    }
  }
  
  return analyzeHybrid(imageDataUrl);
}

// 촬영 버튼 onclick에서 analyzeHybrid 대신 analyzeWithPIIProtection 호출
```

### 설정 옵션

사용자가 PII 블러 켜기/끄기 가능:
```javascript
// 설정 화면에 추가
const PII_BLUR_ENABLED = LS.get('pii_blur') !== 'off'; // 기본 ON

if (PII_BLUR_ENABLED) {
  await analyzeWithPIIProtection(imageDataUrl);
} else {
  await analyzeHybrid(imageDataUrl);
}
```

## 적용 시 주의

### 위험물 UI
- 결과 카드 분기 코드 위치 정확히 찾기 (renderResult 함수)
- 펄스 애니메이션이 너무 강하면 사용자 불편 → 강도 조정
- 빨간색은 색맹 사용자 고려 → 아이콘·텍스트로도 명확히

### PII 블러
- Tesseract.js OCR 정확도 따라 누락 가능성 (특히 손글씨)
- 블러 후 사용자 확인 모달은 매번 뜨면 귀찮음 → 첫 1회 안내 후 기본 ON
- 설정에서 끌 수 있게
- 영수증 외에도 신분증·문서 같이 작동
