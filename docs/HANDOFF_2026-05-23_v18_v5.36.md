### 사고 3 — ⚠️ **`app.html` push까지 간 사고** (가장 큰 사고)
- **시점**: v5.34 작업 중, `confidence` 한 줄 교체 Edit 직후
- **증상**: 파일 끝이 `LS.set('re` 중간에서 잘림. init 함수 닫기 + `init()` 호출 + IIFE `})()` 닫기 + `</script>` + `</body>` + `</html>` 통째로 사라짐 (약 500바이트)
- **발견 시점**: ❌ **push까지 가서 사용자가 "모바일이 헛돈다"고 신고하고 나서야 잡음**
- **사용자 영향**: 모바일에서 10번 새로고침해도 헛돌음. Service Worker 캐시 클리어 + 재push 필요. 사용자 좌절·시간 손실
- **회복**:
  1. `tail -50 app.html` 로 잘린 위치 확인
  2. `grep "async function init"` 로 enclosing 함수 패턴 파악 — `(function() { 'use strict'; ... async function init() { ... } init(); })()` 구조
  3. Python으로 누락 부분 재구성 (init 닫기 → `init();` → `})();` → `</script>` → `</body>` → `</html>`)
  4. `node -c` JS syntax 통과 확인 후 재push
- **이 사고 이후 룰**: push 전 **`tail` + JS syntax + quick_check 3중 검증** 절대 건너뛰지 말 것

### 사고 4 — `app.html` Python write 후 잘림 (재발)
- **시점**: v5.36 작업 중, SYSTEM_PROMPT 컵 구별 룰 추가 Python write 직후
- **증상**: `state.regionLevel2 = level2;` 직후부터 끝까지 잘림
- **발견**: 즉시 검증(`src.rstrip().endswith('</html>')`)에서 잡음
- **회복**: Python으로 다시 끝부분 재구성. **이후 모든 write에 `f.flush() + os.fsync(f.fileno())` 도입**
- **영향**: push 전에 잡음. 사용자 0

### 4번 사고로 정착한 룰 (메모리에 박힘)

`feedback_edit_korean_truncation.md` 메모리에 다음 룰 박힘:

1. **한국어 30+줄 블록은 Edit 대신 Python** (Edit이 multi-byte에서 잘림)
2. **Python `f.write()` 후 항상 `f.flush()` + `os.fsync(f.fileno())`** (마운트 폴더 IO 안전)
3. **write/Edit 직후 즉시 검증 3가지**:
   - `src.rstrip().endswith('</html>')`
   - `node -c` JS syntax
   - `python3 scripts/quick_check.py`
4. **push는 위 3가지 다 통과 후에만**

회복 절차 (오늘 4번 모두 통과한 패턴):
1. `tail -10 app.html` 로 잘림 위치
2. `grep -n "async function init\|(function() {"` 로 enclosing 구조 파악
3. 누락 코드 재구성: if/catch/setTimeout 닫기 → init 함수 닫기 → `init();` → `})();` → html 종료
4. Python `f.write + flush + fsync` 로 저장
5. 검증 3가지 통과 후 진행
