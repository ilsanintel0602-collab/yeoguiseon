# 자동 정책·가격 동기화 시스템 설계 (v5.82+ Opus)

## 배경 (v5.81 사용자 본질 지적)

> "재활용봉투 가격이 자동 업그레이드 되나요? 정책이 바뀔 때마다 금액 변동 등이 자동 반영 안 되면 오해 소지가 된다"

**현재 상태:**
- 정적 JSON (national_rules·bag_prices·region_exceptions) 수동 갱신
- 환경부 정책 변경·봉투 가격 인상 → 우리 앱은 멈춤
- 정직성 위반 위험 (변경된 정책을 옛 안내로)

## 자동 동기화 대상 (우선순위)

### 🔴 P0 (자극적 거짓정보 위험)
1. **자원순환보증금 정책** — 금액·시행 지역·매장 범위
   - 출처: 자원순환보증금관리센터 (cosmo.or.kr)
   - 변동 주기: 환경부 정책 변경 시 (수년·1년 단위)
2. **종량제 봉투 가격** — 지자체별 가격 인상
   - 출처: 261 시군구 공식 사이트 + 환경부 종량제 봉투 통합 정보
   - 변동 주기: 연 1회·물가 인상 시
3. **분리수거 정책 변경** — 카테고리·배출 방법
   - 출처: 환경부 분리배출.kr
   - 변동 주기: 분기·반기

### 🟡 P1
4. **시·구청 연락처 변경**
5. **재활용센터 위치·운영 시간**

## 아키텍처

### 옵션 A: GitHub Actions cron (권장)
```
[매주 일요일 02:00 UTC]
   ↓
[cron job 실행]
   ↓
[① 환경부 분리배출.kr 모니터링]
   ↓ (변경 감지)
[② 자원순환보증금센터 모니터링]
   ↓ (시행 지역·금액 비교)
[③ 시군구 봉투 가격 비교]
   ↓ (가격 변경 감지)
[변경 사항 발견 → GitHub Issue 자동 생성]
   ↓
[사용자 검토 → 수동 또는 자동 PR 머지]
   ↓
[push → 라이브 반영]
```

**장점:**
- 무료 (GitHub Actions 무료 한도)
- 변경 추적 자동
- 사람 검토 단계 보존 (안전)

**단점:**
- 환경부 사이트 구조 변경 시 cron 깨짐 (월 1회 점검 필요)

### 옵션 B: Cloudflare Workers Scheduled (cron)
```
[매일 자정 KST]
   ↓
[Worker fetch 환경부 페이지]
   ↓
[KV 캐시 비교]
   ↓
[변경 감지 → 사용자에게 알림 (SW notify)]
```

**장점:** 더 빠른 감지 (일 단위)
**단점:** Worker 비용 (무료 한도 충분, 단 정기 호출 늘어남)

### 옵션 C: 사용자 신고 + 크라우드 검증
- 사용자가 매장에서 "환급 안 됨" 신고 → 자동 누적
- 임계값 (3회 이상 신고) → 자동 알림
- IndexedDB + Worker 통합

**장점:** 실제 사용 사례 기반 (가장 정확)
**단점:** 사용자 행동 필요

## 본질 가치

✅ **정직성 영구 강화** — 정책 변경 시 자동 인지·반영
✅ **자극적 거짓정보 영구 차단** — "전국 환급" 같은 오안내 위험 0
✅ **사용자 부담 0** — 사용자 행동 없이 자동 갱신
✅ **본질 룰 ⑳ 강화** — policyLastVerified 자동 갱신

## 위험 점검

- **외부 사이트 구조 변경** → cron 깨짐 (월 1회 점검 필수)
- **GitHub Actions 무료 한도** (분 단위) → 주 1회 충분
- **잘못된 자동 변경** → 단계 A는 GitHub Issue만 생성 (자동 PR 머지 X). 사용자 검토 단계 보존
