## 🏗 4 Phase 로드맵

### **Phase A: 인프라 (1주, 자동화 핵심)**
**목표:** 사용자 작업 → 0. 모든 데이터·코드 자동.

1. **Cloudflare D1 DB 마이그레이션** (3시간)
   - `national_rules.json` → D1 테이블
   - `region_exceptions.json` → D1 테이블
   - app.html이 Worker /data API 호출
   - 정적 JSON 폐기 (백업만 유지)

2. **GitHub Actions cron** (2시간)
   - 매주 일요일 자동 크롤링
   - data.go.kr, 환경부, 행안부, 지자체 페이지
   - 자동 검증 + D1 업데이트

3. **자동 검증 파이프라인** (1시간)
   - regions_meta 자동 비교 (오늘 60+ 잘못 매핑 같은 실수 X)
   - JSON schema validation
   - 카테고리 enum 자동 체크

4. **Cloudflare KV 캐싱** (1시간)
   - 이미지 결과 24h 캐시 ← v1.8 ✅
   - 검색 결과 1h 캐시
   - 지역 정보 영구 캐시

**Phase A 완료 기준: 사용자 작업 0건/주, 자동 갱신 매주**

---

### **Phase B: AI 강화 (2주, 정확도 핵심)**
**목표:** 95%+ 정확도, 단일 모델 의존 X.

1. **앙상블 모델** (1주)
   - Gemini Flash (기본) ✅
   - Claude Haiku 4.5 (폴백) ← v1.8 ✅
   - 자체 CNN (TensorFlow.js, 흔한 품목 즉시 분류)
   - 결과 신뢰도 비교 → 최선 선택

2. **벡터 검색** (3일)
   - Pinecone 또는 Weaviate (무료층)
   - 한국어 임베딩 (multilingual-e5)
   - "노트북" → semantically 비슷한 item 매칭
   - Gemini 의존 ↓

3. **사용자 정정 → 자동 학습** (3일)
   - 사용자 👍/👎 → Worker KV
   - 매주 자동 분석 → 약점 카테고리 식별
   - alias 자동 보강 (현재 augment_v2 인프라 활용)

4. **벤치마크 자동화** (1일)
   - TrashNet 102장 매주 자동 평가
   - 카테고리별 정확도 추적
   - 점수 하락 시 자동 알림

**Phase B 완료 기준: TrashNet 95%+, 사용자 정정 자동 반영**

---

### **Phase C: UX 차별점 (1주, 한국 사용자 경험)**
**목표:** 한국 사용자에게 *친절·정확·빠른* 분리수거 경험 제공.

1. **GPS + 지도** (3일) ⭐ 핵심
   - 사용자 GPS → 가까운 수거함 자동 (행안부 데이터)
   - Naver/Kakao Maps SDK
   - 길찾기 링크 자동 생성

2. **음성 입력** (2일) ⭐ 한국어 손 더러울 때
   - Web Speech API
   - 한국어 우선

3. **결과 공유** ← v5.26 ✅
   - 카톡·문자 공유

4. **첫 사용자 onboarding** (1일)
   - 30초 안내 모달
   - 이탈률 ↓

**Phase C 완료 기준: GPS 기반 수거함 안내, 한국어 음성 입력, onboarding**

*다국어는 우선순위 낮음 (Phase D Optional).*

---

### **Phase D: 운영 (지속)**
**목표:** 자동 발견 → 자동 보강 사이클.

1. **모니터링** (1일)
   - Sentry 에러 트래킹
   - Cloudflare Analytics
   - 사용자 행동 분석

2. **A/B 테스트** (1일)
   - 새 UI 변경 시 자동 A/B
   - 데이터 기반 결정

3. **자동 보강 사이클** (지속)
   - 매주 약점 카테고리 자동 식별
   - augment_v2 자동 재실행
   - 자동 검증 + 자동 push

4. **커뮤니티** (선택)
   - 사용자가 새 룰 제안 → 검토 → 자동 통합
