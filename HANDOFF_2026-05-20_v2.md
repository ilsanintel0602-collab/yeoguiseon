### 🔴 P0 — Gemini 빈 응답 해결 (10~15분)

**증상** (모바일 v5.6.3 테스트 시):
- toast: "⚠️ Gemini 실패: Gemini 응답 비어있음 → COCO-SSD 폴백"
- 페트병 같은 흔한 물건은 COCO-SSD 폴백으로도 잘 작동 (94%)
- 유리컵·종이컵·영수증 등 **어려운 케이스는 폴백이 못 잡음** → 정확도 떨어짐

**원인 가장 유력**: Worker 코드의 `generationConfig`에 `safetySettings` 명시 안 됨 → Gemini 기본 safety filter (medium)가 "위험물·주사기·리튬배터리·깨진 유리" 같은 프롬프트 단어에 반응해 응답 차단.

**해결 패치** (`scripts/cloudflare_worker.js` line 110~121):
```js
const payload = {
  contents: [{ parts: [...] }],
  generationConfig: {
    responseMimeType: "application/json",
    temperature: Number.isFinite(body.temperature) ? body.temperature : 0.2,
  },
  // ★ 추가 ★
  safetySettings: [
    { category: "HARM_CATEGORY_HARASSMENT", threshold: "BLOCK_NONE" },
    { category: "HARM_CATEGORY_HATE_SPEECH", threshold: "BLOCK_NONE" },
    { category: "HARM_CATEGORY_SEXUALLY_EXPLICIT", threshold: "BLOCK_NONE" },
    { category: "HARM_CATEGORY_DANGEROUS_CONTENT", threshold: "BLOCK_NONE" },
  ],
};
```

추가로 디버그용 — Worker 응답에 `finishReason` 포함:
```js
const candidate = geminiData?.candidates?.[0];
const text = candidate?.content?.parts?.[0]?.text ?? "";
const finishReason = candidate?.finishReason || "unknown";
// json 응답에 finishReason 함께 전달 → app.html에서 toast로 표시
return json({ ok: true, result: parsed ?? text, finishReason }, 200, ...);
```

**사용자 작업**: Cloudflare 대시보드 → yeoguiseon-proxy → Edit code → 위 패치 붙여넣기 → Save and Deploy. 5~10분.
