# 데이터 검증 스킬

regions.json과 app.html의 RULES 데이터를 공식 출처와 비교해서 정확성을 검증.

## 언제 사용?

- 사용자가 "데이터 검증해줘" 요청할 때
- 정기적인 분기별 점검
- 사용자 신고가 누적됐을 때
- 새 지역 추가 후 품질 검사

## 처리 단계

### 1단계: 현재 데이터 로드
- regions.json 읽기
- 각 지역의 officialSource URL 확인

### 2단계: 공식 출처 fetch
- 각 지자체의 officialSource WebFetch
- 환경부 '내손안에 분리배출' 데이터 비교
- 변경된 날짜 확인 (lastUpdated와 비교)

### 3단계: 항목별 비교

각 item × 각 region 매트릭스에 대해:
- 현재 데이터의 category가 공식 자료와 일치?
- note가 최신 정보인가?
- 출처 URL 여전히 작동하는가?

### 4단계: confidence 자동 조정

| 상황 | confidence 변경 |
|---|---|
| 공식 자료와 일치 | high 유지 |
| 부분 일치 (요일만 다름 등) | medium |
| 불일치 발견 | low + "확인 필요" 플래그 |
| 공식 자료 URL 404 | low + 출처 재확인 필요 |

### 5단계: 변경 사항 보고

```
검증 보고 (2026-XX-XX)
=====================
검증한 지역: 강남구, 일산서구
검증한 물건: 20개

✅ 일치 (변경 없음): 35개 항목
⚠️ 부분 일치 (note 갱신): 3개 항목
   - 강남구 × pet_bottle: 배출 요일이 목→수로 변경
❌ 불일치 발견: 2개 항목
   - 일산서구 × 코팅 종이컵: 일반→재활용 변경 가능성

권장 조치:
- regions.json 업데이트
- app.html 동기화
- 사용자에게 변경 안내 (앱 내 공지)
```

### 6단계: 자동 업데이트 (사용자 승인 후)
- regions.json 업데이트
- app.html 인라인 데이터 동기화
- PROJECT_STATUS.md에 변경 이력 추가

## 안전 가드레일

- ❌ 사용자 승인 없이 자동 적용하지 말 것
- ✅ 변경 사항은 항상 diff로 보여줄 것
- ✅ 공식 출처 외 (블로그, 뉴스)는 참고만, 직접 적용 금지
- ✅ 의심스러우면 confidence를 보수적으로 낮출 것

## 예시 사용

사용자: "전체 데이터 검증해줘"

스킬 실행:
1. regions.json 로드 (2개 지역)
2. clean.gangnam.go.kr fetch
3. www.goyang.go.kr fetch
4. 환경부 자료 비교
5. 20개 물건 × 2개 지역 = 40개 셀 검증
6. 변경 사항 diff 보고
7. 사용자 승인 대기
8. 승인 후 업데이트 + PROJECT_STATUS 기록
